# Checklist — caf-operations-management

- [ ] Operations maturity level assessed (baseline / enhanced / business-aligned)
- [ ] Resource inventory complete and up-to-date
- [ ] All resources have health probes or availability monitoring
- [ ] Metrics collection enabled per service (CPU, memory, requests, errors)
- [ ] Structured logging configured (Application Insights or Log Analytics)
- [ ] Distributed tracing enabled for request flows
- [ ] Alert rules defined for critical metrics (availability, error rate, latency)
- [ ] Alert routing configured (email, Slack, PagerDuty)
- [ ] Backup configuration verified for all data stores
- [ ] Backup retention matches policy (7 days dev, 30 days prod)
- [ ] Backup restore tested within last 90 days
- [ ] Update management strategy defined (ACA image updates, PG minor versions)
- [ ] SLA/SLO targets defined per workload
- [ ] SLA measurement methodology documented (uptime calculation, error budget)
- [ ] Operational runbooks documented for common procedures
- [ ] Incident response procedure documented and tested
- [ ] Operational fitness review conducted
- [ ] Operations improvement roadmap created
- [ ] Evidence captured in `docs/evidence/{stamp}/caf/operations/`
