# Release Records (SSOT)

Per-release YAML files stored as `<tag>.yaml`.

## Generation

Generated by `scripts/build_release_manifest.py --yaml`:

```bash
python scripts/build_release_manifest.py \
  --tag prod-20260303-1200 \
  --image-digest sha256:abc123... \
  --image-tag prod-20260303-1200 \
  --commit-sha abc123def456... \
  --gates G1,G2,G3 \
  --deployer github-actor \
  --run-id 12345 \
  --yaml
```

## Rules

- **Append-only**: Each deployment creates a new `<tag>.yaml` file.
- **Never hand-edit**: Always regenerate via `build_release_manifest.py`.
- **Contract**: All records conform to `ssot/runtime/release_contract.yaml`.
- **Validation**: The script validates required fields before writing.

## Schema

Each record contains the fields defined in `release_contract.yaml`:

| Field | Required | Description |
|-------|----------|-------------|
| `tag` | yes | Immutable git tag |
| `image_digest` | yes | Docker image digest (sha256) |
| `image_tag` | yes | Docker image tag |
| `commit_sha` | yes | Full 40-char commit SHA |
| `addon_set_fingerprint` | yes | Hash of addon manifest list |
| `sbom_artifact` | yes | SBOM file path (null if unavailable) |
| `gates_passed` | yes | Gates passed (G1-G4) |
| `deployer` | yes | GitHub actor |
| `workflow_run_id` | yes | GitHub Actions run ID |
| `timestamp` | yes | UTC ISO8601 timestamp |
| `vuln_scan_result` | no | Vulnerability scan summary |
| `oca_lock_hash` | no | Hash of oca.lock.json |
