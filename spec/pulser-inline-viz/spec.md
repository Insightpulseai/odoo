# Spec — Pulser Inline Visualization Engine
`.specify/spec.md`
Rev: 2026-04-15 | Status: READY FOR PLAN

---

## Overview

Pulser agents emit a `viz_payload` JSON object alongside their text answer.
A renderer service consumes the payload and returns a render artifact:
- Teams: Adaptive Card JSON (rendered by Teams client natively)
- Odoo: QWeb HTML string (injected into chatter message body)
- ops-console: Chart.js React component (rendered inline in chat panel)

Chart images (PNG/SVG) are generated by Foundry Code Interpreter,
stored in `stipaidevagent`, and linked in the payload.

## F-001 — viz_payload schema

**Purpose:** Single contract between agent and renderer.

```jsonc
{
  "$schema": "https://ipai.internal/viz-payload/v1",
  "id": "<uuid>",                      // idempotency key
  "agent": "bank_recon",               // originating agent name
  "title": "Reconciliation Summary",   // card/chart title
  "subtitle": "Apr 2026 · TBWA\\SMP",  // sub-label
  "viz_type": "kpi_cards",             // see §F-002
  "kpi_cards": [...],                  // present when viz_type=kpi_cards
  "chart": {...},                      // present when viz_type=chart
  "insight_pills": [...],              // optional — always present in agency use cases
  "data_source": "odoo:account_move",  // lineage
  "generated_at": "2026-04-15T13:00Z",
  "blob_url": null                     // set by renderer after Code Interpreter runs
}
```

**Acceptance criteria:**
- [ ] JSON Schema draft-07 file at `platform/contracts/viz-payload.schema.json`
- [ ] Schema rejects unknown `viz_type` values
- [ ] Schema rejects missing required fields
- [ ] Schema validated in CI via `ajv` CLI before any renderer code runs
- [ ] Contract test: `tests/contract/test_viz_payload_schema.py`

## F-002 — viz_type catalogue

| `viz_type` | Description | Teams | Odoo | Console |
|---|---|---|---|---|
| `kpi_cards` | 1–6 metric tiles (value + delta + label) | FactSet | Table | Card grid |
| `line_chart` | Time-series trend (1–3 series) | PNG via blob | Chart.js | Chart.js |
| `bar_chart` | Categorical comparison (1–3 series) | PNG via blob | Chart.js | Chart.js |
| `grouped_bar` | Multi-series bar (e.g. SOV by channel) | PNG via blob | Chart.js | Chart.js |
| `donut_chart` | Market share / composition | PNG via blob | Chart.js | Chart.js |
| `sparkline` | Inline mini trend (no axes) | PNG via blob | SVG inline | SVG inline |
| `insight_pills` | Text badges: win/risk/neutral | Fact rows | `<span>` pills | Pill row |

## F-003 — Agent viz_payload emission

**Purpose:** All 6 agents emit `viz_payload` when their response includes quantitative data.

Rules:
1. If response contains ≥1 KPI figure → emit `kpi_cards` payload
2. If response contains trend/time-series data → emit `line_chart` or `sparkline`
3. If response contains comparison data (brand A vs B, period vs period) → emit `bar_chart` or `grouped_bar`
4. If response contains composition/share data → emit `donut_chart`
5. Payload is emitted as a separate structured output alongside the text answer
6. Agent never blocks waiting for render — emit payload, move on; renderer is async

**Per-agent primary chart types:**

| Agent | Primary viz | Data source |
|---|---|---|
| Bank Recon | `kpi_cards` + `line_chart` (reconciled vs unreconciled) | `account_move`, `account_bank_statement_line` |
| Finance Close | `kpi_cards` + `bar_chart` (budget vs actual) | `mis_builder`, `account_analytic_line` |
| Tax Guru | `kpi_cards` + `insight_pills` | `ipai_bir_filing_run` |
| AP Invoice | `kpi_cards` + `grouped_bar` (first-pass-yield by vendor) | `purchase_order`, `account_move` |
| Doc Intel | `kpi_cards` (OCR confidence) + `insight_pills` | `docai-ipai-dev` telemetry |
| Pulser | All types depending on query | All sources |
| Ces (Ask Ces) | `grouped_bar` (SOV), `kpi_cards`, `insight_pills` | `ces_campaign_performance` |
| Scout (Suqi) | `line_chart` (volume trend), `donut_chart` (brand share) | `scout_daily_summary`, `scout_brand_share` |

**Acceptance criteria:**
- [ ] Each agent has a `build_viz_payload()` function in its skill module
- [ ] Function accepts agent response dict, returns `VizPayload` dataclass or `None`
- [ ] Unit tests: `tests/unit/test_{agent}_viz_payload.py` for all 8 agents
- [ ] Payload schema validated before emission

## F-004 — Foundry Code Interpreter renderer

**Purpose:** Generate PNG/SVG charts for targets that cannot render Chart.js natively (Teams).

Flow:
```
viz_payload (chart type) 
  → Code Interpreter prompt: "Generate a {type} chart with this data as PNG, return base64"
  → base64 PNG
  → upload to stipaidevagent blob container "charts"
  → blob URL injected into viz_payload.blob_url
  → payload forwarded to Teams / Odoo / console renderer
```

Foundry resource: `ipai-copilot-resource`, project `ipai-copilot`, East US 2
Model for Code Interpreter: `gpt-5.3-codex`
Storage: `stipaidevagent`, container `charts`, path `{agent}/{date}/{id}.png`
AFD served URL: `https://erp.insightpulseai.com/charts/{agent}/{date}/{id}.png`

**Acceptance criteria:**
- [ ] `agent-platform/renderers/foundry_renderer.py` — CLI: `pulser-viz render-chart`
- [ ] Accepts `viz_payload` JSON via stdin, returns blob URL via stdout
- [ ] Chart style matches IPAI design tokens (TBWA black #1A1A1A, yellow #FFD100, teal #1D9E75)
- [ ] Renders all 5 chart types (line, bar, grouped_bar, donut, sparkline) correctly
- [ ] PNG dimensions: 800×400px for charts, 400×200px for sparklines
- [ ] Contract test: verifies blob URL is reachable via AFD
- [ ] p95 render time: < 8 seconds (measured in integration test)

## F-005 — Teams Adaptive Card renderer

**Purpose:** Render `viz_payload` as Teams Adaptive Card v1.5.

Card anatomy by viz_type:

**`kpi_cards`:**
```json
{
  "type": "AdaptiveCard",
  "version": "1.5",
  "body": [
    { "type": "TextBlock", "text": "{title}", "weight": "Bolder" },
    { "type": "TextBlock", "text": "{subtitle}", "size": "Small", "isSubtle": true },
    { "type": "ColumnSet", "columns": [
      { "type": "Column", "items": [
        { "type": "TextBlock", "text": "{label}", "size": "Small", "isSubtle": true },
        { "type": "TextBlock", "text": "{value}", "size": "Large", "weight": "Bolder" },
        { "type": "TextBlock", "text": "{delta}", "color": "Good|Warning|Attention" }
      ]}
    ]}
  ]
}
```

**Chart types:** `Image` element with `url: blob_url` from F-004.

**`insight_pills`:** `FactSet` element with colored indicator text.

**Acceptance criteria:**
- [ ] `agent-platform/renderers/teams_renderer.py` — CLI: `pulser-viz render --target teams`
- [ ] Accepts `viz_payload` JSON via stdin, returns Adaptive Card JSON via stdout
- [ ] All 7 viz_types produce valid Adaptive Card JSON
- [ ] Adaptive Card validated against Teams schema validator CLI
- [ ] Renders correctly in Teams Desktop, Web, and Mobile (manual smoke test)
- [ ] Delta values use correct color: up+positive=Good, up+negative=Warning, down+negative=Attention
- [ ] Unit tests: `tests/unit/test_teams_renderer.py`

## F-006 — Odoo chatter QWeb renderer

**Purpose:** Render `viz_payload` as HTML injected into Odoo chatter message body.

Module: `ipai_viz_chatter` (new delta module, depends on `mail`)
Approach: QWeb template + Chart.js loaded from `https://cdnjs.cloudflare.com`

Template structure:
```xml
<t t-name="ipai_viz_chatter.kpi_cards">
  <div class="ipai-viz-kpi-grid">
    <t t-foreach="payload['kpi_cards']" t-as="kpi">
      <div class="ipai-kpi-card">
        <span class="ipai-kpi-label"><t t-esc="kpi['label']"/></span>
        <span class="ipai-kpi-value"><t t-esc="kpi['value']"/></span>
        <span t-attf-class="ipai-kpi-delta #{kpi['direction']}">
          <t t-esc="kpi['delta']"/>
        </span>
      </div>
    </t>
  </div>
</t>

<t t-name="ipai_viz_chatter.chart">
  <div class="ipai-viz-chart">
    <img t-att-src="payload['blob_url']" alt="Chart" style="max-width:100%;border-radius:8px;"/>
  </div>
</t>
```

**Acceptance criteria:**
- [ ] `odoo/addons/ipai/ipai_viz_chatter/` module structure (CE conventions)
- [ ] `<list>` views only (never `<tree>`)
- [ ] Chart.js loaded from cdnjs CDN — no local bundle
- [ ] QWeb templates for all 7 viz_types
- [ ] `ir.model.access.csv` with correct permissions
- [ ] Manual render test: Bank Recon agent posts KPI card to Odoo chatter
- [ ] Module installs cleanly on `ipai-odoo-dev` (ACA)
- [ ] Unit tests via Odoo test runner: `tests/test_viz_chatter.py`

## F-008 — Async render pipeline (Service Bus)

**Purpose:** Agent emits payload to queue; renderer picks up asynchronously so agent
response is never blocked by chart generation.

Flow:
```
Agent emits viz_payload
  → POST to Service Bus topic "viz-render" (sb-ipai-dev-sea)
  → Agent returns text response immediately (< 1 second)
  → Viz renderer worker (ACA Job: ipai-viz-renderer-job)
      → dequeues payload
      → calls Foundry Code Interpreter if chart type
      → uploads PNG to stipaidevagent
      → POSTs blob URL back to originating session via cosmos-ipai-dev
  → Client (Teams/Odoo/console) polls cosmos for updated payload (5s interval)
  → Renders inline when blob_url available
```

**Acceptance criteria:**
- [ ] `ipai-viz-renderer-job` ACA Job created (Bicep: `infra/azure/agent/viz-renderer-job.bicep`)
- [ ] Service Bus topic: `viz-render`, subscription: `viz-renderer-worker`
- [ ] Dead-letter queue: `viz-render-dlq` — failed renders after 3 retries
- [ ] Idempotency: duplicate payloads (same `id`) deduplicated in worker
- [ ] `ops.agent_runs` row created per render job (Odoo)
- [ ] `cosmos-ipai-dev` document updated with `blob_url` on completion
- [ ] Integration test: end-to-end render via queue, verify blob URL reachable
