# Azure Secret & Key Vault Security Audit

**Date**: 2026-03-14
**Scope**: All Azure Key Vaults, Container App identities, env var injection, repository secret leakage
**Assessed by**: Security Engineering (automated + manual)
**Security Score**: **62 / 100**

---

## 1. Summary

The InsightPulse AI Azure environment uses Azure Key Vault as the canonical secret store across three environments (dev, staging, prod) plus two special-purpose vaults. RBAC is well-structured with group-based access, break-glass accounts, and managed identities for most Container Apps.

However, six CRITICAL secret leaks exist in the repository (committed credentials), three Container Apps lack managed identity assignment, and eleven secrets required by integrations are missing from Key Vault. The most urgent action is rotating the six leaked credentials and purging them from git history.

| Category | Status |
|----------|--------|
| Key Vault structure | Well-organized, multi-environment |
| RBAC | Groups + break-glass in place |
| Managed identities | 8/11 Container Apps assigned |
| Repository leakage | 6 CRITICAL, 4 HIGH findings |
| Secret completeness | 11 expected secrets missing from KV |
| Default passwords | Present in 3 tracked config files |

## 8. Recommendations (Prioritized)

### P0 -- Immediate (Within 24 Hours)

| # | Action | Rationale |
|---|--------|-----------|
| 1 | **Rotate all 6 CRITICAL leaked secrets** (Google OAuth, Gemini, Supabase service role, n8n JWT, webhook HMAC, Odoo master password) | These credentials are compromised. Rotation is non-negotiable. |
| 2 | **Run `git filter-repo` or BFG to purge secrets from git history** | Removing files is insufficient; secrets persist in git history. |
| 3 | **Add pre-commit secret scanning** (e.g., `detect-secrets`, `gitleaks`) to CI pipeline | Prevent future leaks at the commit stage. |
| 4 | **Remove `config/prod/odoo.conf` from tracking** and add to `.gitignore` | Production config with `db_password=odoo` must not be in version control. |

### P1 -- This Week

| # | Action | Rationale |
|---|--------|-----------|
| 5 | **Enable managed identity on ipai-shelf-dev, ipai-auth-dev, ipai-superset-dev** | These 3 apps cannot authenticate to Key Vault without identity. |
| 6 | **Provision missing secrets** (slack-bot-token, github-token, plane-api-key, openai-api-key, stripe-secret-key, odoo-admin-password) into kv-ipai-dev | Integration secrets must be centralized in Key Vault. |
| 7 | **Migrate Google OAuth credentials** to Key Vault after rotation | Replace hardcoded values with secretRef injection. |
| 8 | **Move ACR credentials** from ipai-odoo-dev-kv to kv-ipai-dev (shared vault) | All Container Apps need registry access, not just Odoo. |

### P2 -- This Sprint

| # | Action | Rationale |
|---|--------|-----------|
| 9 | **Replace default passwords** in Docker Compose and Dockerfile.ci with environment variable references | Default `:-odoo` and `:-admin` fallbacks are insecure in non-local contexts. |
| 10 | **Remove CHANGE_ME placeholders** and replace with fail-fast env var checks | Placeholders may accidentally reach production. |
| 11 | **Audit kv-ipai-prod and kv-ipai-staging** with the same methodology | Staging and prod vaults were not enumerated; they may have gaps. |
| 12 | **Move USER (odoo_admin) from plaintext to secretRef** in Container App env vars | Usernames are low-risk but should follow the principle of least exposure. |

### P3 -- Next Sprint

| # | Action | Rationale |
|---|--------|-----------|
| 13 | **Enable Key Vault secret rotation policies** for database passwords and API keys | Automated rotation reduces exposure window for compromised credentials. |
| 14 | **Consolidate OpenAI API key** from Supabase vault into Azure Key Vault | Split secret stores create drift and audit blind spots. |
| 15 | **Implement Key Vault access logging** via Azure Monitor diagnostic settings | Audit trail for secret access is required for compliance. |
| 16 | **Document secret ownership** -- assign each secret a responsible team/person | Unowned secrets do not get rotated. |
