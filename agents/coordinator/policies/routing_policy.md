# Routing Policy

## git-expert

Route when the request involves:

- branching / rebasing / merges, release strategy, tags
- bisect/blame/reflog, conflict resolution
- commit conventions and git hygiene automation

## devops-expert

Route when the request involves:

- GitHub Actions, CI failures, caches, artifacts, releases
- Supabase/Vercel/Cloudflare/DigitalOcean deploy or env parity
- secrets, OIDC, rollback, health checks, observability

## repo-expert

Route when the request involves:

- repo restructuring, boundaries, directory conventions
- Spec Kit compliance, SSOT doc drift, CODEOWNERS
- “find X in repo” / reuse prior art / governance validators
