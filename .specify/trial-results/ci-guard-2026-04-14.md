## Summary

CI Guard v1.0.0 is structurally incompatible with our pinned spec-kit v0.6.2. The extension's command names use the legacy `speckit.ci.*` namespace, but spec-kit v0.6.2 enforces extension-ID namespacing (`ci-guard.*`). Install fails at the validation step before any files are written.

**Trial Gate 1 (zero overwrites of existing `.claude/commands/speckit.*`):** ✅ Passed by virtue of install failure — atomic abort, no state mutation.
**Trial Gates 2–4:** Not reached.

## Root cause

The upstream extension was authored against spec-kit < v0.5.x, when the `speckit.*` global command namespace was open to extensions. Spec-kit v0.5+ tightened to require extensions to namespace commands under their extension ID. Author's last `updated_at` is 2026-04-10 — the v1.0.0 release predates or ignores this validation tightening.
