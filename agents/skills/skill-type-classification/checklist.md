# Skill Type Classification — Checklist

- [ ] Skill purpose is understood
- [ ] Mechanism assessed (model enhancement vs. process encoding)
- [ ] Classification applied: capability_uplift or encoded_preference
- [ ] Eval strategy assigned: regression testing or fidelity validation
- [ ] Durability flagged: medium or high
- [ ] Classification recorded in skill-contract.yaml field `skill_classification`
- [ ] Reassessment trigger documented
