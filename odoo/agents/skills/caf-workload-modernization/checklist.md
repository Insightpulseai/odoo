# Checklist — caf-workload-modernization

- [ ] Current workload architecture documented with technology stack
- [ ] Performance baselines captured (response time, throughput, resource utilization)
- [ ] Modernization opportunities identified and prioritized
- [ ] Modernization pattern selected per workload (refactor, rearchitect, rebuild)
- [ ] Target architecture designed with Azure PaaS services
- [ ] Containerization readiness assessed (if not already containerized)
- [ ] Database modernization options evaluated
- [ ] Incremental modernization path defined (strangler fig or parallel run)
- [ ] Cost impact estimated (migration effort + runtime cost change)
- [ ] Performance improvement targets defined with measurable metrics
- [ ] Integration points and dependencies accounted for
- [ ] Rollback strategy defined for each modernization stage
- [ ] Modernization executed in stages with validation gates
- [ ] Post-modernization performance measured and compared to baseline
- [ ] Cost actuals compared to estimates
- [ ] Evidence captured in `docs/evidence/{stamp}/caf/modernization/`
