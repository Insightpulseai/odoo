# caf-cloud-native-design

Designs new cloud-native workloads using CAF Cloud-native phase methodology, covering microservices, container orchestration, serverless patterns, event-driven design, and DevOps practices.

## When to use
- New greenfield workload design
- Serverless or event-driven architecture evaluation
- Microservices decomposition assessment
- Container orchestration platform selection
- Cloud-native data service selection

## Key rule
ACA is the default compute platform. AKS requires explicit justification. Microservices decomposition must be justified by team size and scale requirements, not architectural fashion.

## CAF Phase
Cloud-native
