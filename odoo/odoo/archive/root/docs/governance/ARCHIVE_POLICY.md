# Archive Policy

## Purpose

Archived repositories remain part of org history, but they are not active delivery surfaces.

## Archive rules

A repo may be archived only if:
- its canonical logic has been migrated, or
- it is formally declared historical/non-canonical

## Required archive metadata

Every archived repo must record:
- why it was archived
- what replaced it
- whether any canonical logic remains inside it
- who approved the archive decision

## Prohibited state

Do not leave a repo archived if it still contains:
- active automation dependencies
- current deployment logic
- current product SSOT
- undocumented canonical code
