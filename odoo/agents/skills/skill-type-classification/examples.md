# Skill Type Classification — Examples

## Example 1: odoo-orm-model-extension → encoded_preference

- The model CAN write Odoo ORM code without this skill
- This skill enforces our specific attribute ordering, naming conventions, and patterns
- **Classification**: encoded_preference
- **Eval**: Fidelity validation — does output match our conventions?
- **Durability**: High — conventions don't change when models improve

## Example 2: foundry-model-selection → capability_uplift

- Without this skill, the model may not know current Foundry model catalog details
- The skill provides current benchmark data and selection criteria
- **Classification**: capability_uplift
- **Eval**: Regression testing — does the skill still provide value over base model?
- **Durability**: Medium — base models may learn Foundry catalog natively

## Example 3: agent-pattern-selection → encoded_preference

- The model CAN choose patterns without this skill
- This skill encodes Anthropic's specific simplicity-first doctrine
- **Classification**: encoded_preference
- **Eval**: Fidelity validation — does it follow the hierarchy correctly?
- **Durability**: High — the doctrine is a deliberate choice
