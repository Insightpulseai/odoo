# Security & Governance

## Identity

- **Target IdP**: Microsoft Entra ID (greenfield — migration from Keycloak not started)
- **Current**: Basic login per app (no SSO operational)
- **Agent identity**: Entra Agent ID (Preview) for AI agent lifecycle

## Secrets Management

All secrets stored in **Azure Key Vault** (`kv-ipai-dev`):
- Runtime binding via managed identity → Key Vault → env vars
- Never hardcoded, never committed, never echoed in logs

## Network

- **Edge**: Azure Front Door (TLS termination, WAF)
- **DNS**: Cloudflare (authoritative, DNS-only mode for AFD-backed records)
- **Internal**: Azure Container Apps environment (private VNet)

## Compliance

### BIR (Philippines)
- VAT (2550M/2550Q), EWT (1601-E), Compensation WT (1601-C)
- TaxPulse-PH sub-agent for pre-filing validation
- Document Intelligence for BIR Official Receipt extraction

### Data Governance
- Unity Catalog for data access control and lineage
- Row-level security in Supabase for external-facing data
- Audit logging via Odoo `auditlog` module
