# Checklist — caf-adoption-planning

- [ ] Digital estate inventory completed (all servers, databases, applications)
- [ ] Workload dependencies mapped
- [ ] Each workload classified by priority (P0-P3) and complexity (low/medium/high)
- [ ] Migration/modernization waves sequenced with dependency awareness
- [ ] Wave 1 is a quick win with low risk and high visibility
- [ ] Team skills assessed against Azure competency matrix
- [ ] Skills gaps identified with specific training/certification recommendations
- [ ] Organizational alignment model defined (CCoE, cloud team, RACI)
- [ ] Adoption backlog created with prioritized work items
- [ ] Iteration cadence established (sprint length, review frequency)
- [ ] Review gates defined at wave boundaries
- [ ] Success metrics defined per wave (measurable, time-bound)
- [ ] Budget constraints documented and reflected in plan
- [ ] Risk register updated with planning-phase risks
- [ ] Plan integrated with existing planning tools (AzDO Boards, GitHub)
- [ ] Evidence captured in `docs/evidence/{stamp}/caf/planning/`
