IPAI Sample Metrics

Odoo 18 CE OCA Style API Ready

A minimal, OCA-compliant example module demonstrating best practices for Odoo 18 CE custom app development. Perfect as a template for new IPAI modules.

Features

  • Complete data model with computed fields
  • List, Form, and Search views (Odoo 18 syntax)
  • Security ACL configuration
  • Demo/seed data
  • Comprehensive unit tests
  • API-ready helper methods (XML-RPC/JSON-RPC)
  • OCA-style code organization

Data Model

The ipai.sample.metric model tracks analytics metrics with:

  • name: Human-readable metric label
  • code: Technical metric code (e.g., CONV_RATE)
  • date: Metric date
  • brand_id: Related brand (res.partner)
  • store_id: Related store (res.partner)
  • value: Numeric metric value
  • unit: Unit type (%, Count, Amount)
  • is_alert: Auto-computed alert flag

API Usage

External systems can interact via XML-RPC/JSON-RPC:

# Create metric
metric_id = models.execute_kw(
    DB, uid, PASSWORD,
    "ipai.sample.metric", "create_from_payload",
    [{"name": "Conversion Rate", "code": "CONV_RATE", "value": 42.0, "unit": "percent"}],
)

# Read metrics
metrics = models.execute_kw(
    DB, uid, PASSWORD,
    "ipai.sample.metric", "get_metrics",
    [[["code", "=", "CONV_RATE"]]],
)

Installation

  1. Place module in addons/ipai/ipai_sample_metrics/
  2. Update module list in Odoo
  3. Install "IPAI Sample Metrics" from Apps
