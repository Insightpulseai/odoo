- Treat this area as external SDK/bridge/runtime scope.
- Keep Odoo transport abstractions outside addons — this is where they belong.
- Prefer typed interfaces, normalized errors, and stable public APIs.
- Assume Odoo-facing code must survive transport/version change (18 RPC → 19 JSON-2 → 20 JSON-2 only).
- SDK packages must not import from `odoo` namespace — they are external clients.
- Use `httpx` for HTTP, `pydantic` for typing, `tenacity` for retry.
- Reference: `spec/odoo-external-sdk/prd.md` for package architecture.
