# T001 — viz_payload JSON Schema + CI Gate
**Spec ref:** F-001

**Successor:** T002

**ADO Epic:** Shared SaaS Control Plane

## Objective
Publish the canonical `viz_payload` JSON Schema and add a CI gate that
validates all test payloads before any implementation code is merged.

## Tasks

- [ ] Create `platform/contracts/viz-payload.schema.json` (JSON Schema draft-07)
  - Required: `id`, `agent`, `title`, `viz_type`, `generated_at`
  - `viz_type` enum: `kpi_cards`, `line_chart`, `bar_chart`, `grouped_bar`,
    `donut_chart`, `sparkline`, `insight_pills`
  - Conditional: `kpi_cards` array required when `viz_type=kpi_cards`
  - Conditional: `chart` object required for all chart types
  - Optional: `subtitle`, `insight_pills`, `blob_url`

- [ ] Create `tests/contract/test_viz_payload_schema.py`
  - Valid payloads for all 7 viz_types pass
  - Payloads missing required fields fail
  - Unknown `viz_type` values fail

- [ ] Add schema validation step to `azure-pipelines/viz-renderer.yml`:
  ```yaml
  - script: |
      npm install -g ajv-cli
      ajv validate -s platform/contracts/viz-payload.schema.json \
        -d tests/fixtures/viz_payloads/*.json
    displayName: 'Validate viz_payload schema'
  ```

- [ ] Create `tests/fixtures/viz_payloads/` with one valid JSON per viz_type (7 files)

## Done when
- `ajv validate` passes on all fixture files in CI
- Schema rejects at least 5 invalid payloads in contract tests
- PR merged to `main` with pipeline green
- ADO work item linked: `AB#<id>`
