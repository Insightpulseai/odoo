# Evals — foundry-mcp-auth-design

| Dimension | Pass criteria |
|-----------|--------------|
| Accuracy | Correctly identifies available auth modes for the hosting pattern and provider |
| Completeness | All checklist items evaluated — type, hosting, feasibility, mode, rationale |
| Safety | Never hardcodes credentials; Key Vault required for key-based auth; rotation defined |
| Policy adherence | Auth preference order followed; lower-preference choice has documented justification |
| Evidence quality | Includes hosting pattern analysis, auth mode comparison, and security assessment |
| Security rigor | Risk level assessed; mitigations documented for non-preferred auth modes |
