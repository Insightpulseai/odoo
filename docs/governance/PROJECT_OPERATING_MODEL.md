# GitHub Projects Operating Model

## Standard projects

The organization standard is three org-level GitHub Projects:

1. Portfolio / Roadmap
2. Execution Board
3. Ops / Release Readiness

## Portfolio / Roadmap
Use for:
- initiatives
- quarterly roadmap
- strategic priorities
- cross-repo outcomes

## Execution Board
Use for:
- active implementation work
- sprint/cycle execution
- bugs, tasks, features
- PR-linked delivery

## Ops / Release Readiness
Use for:
- release gating
- production readiness
- migrations
- environment checks
- deployment evidence

## Standard fields

- Domain
- Type
- Priority
- Target Quarter
- Sprint/Cycle
- Environment
- Release
- Owner
- Status
- Blocked
- Repo

## Standard item types

- Initiative
- Epic
- Feature
- Task
- Bug
- Risk

## Rule of use

- Figma / FigJam owns planning and visual roadmap design
- GitHub Projects owns approved execution
- Releases and environments provide delivery evidence
