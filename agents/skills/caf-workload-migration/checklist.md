# Checklist — caf-workload-migration

- [ ] Workload readiness assessed (compatibility, sizing, dependencies)
- [ ] Dependency map created (network, data, identity, DNS)
- [ ] Migration pattern selected (rehost or replatform) with justification
- [ ] Migration wave sequenced with dependency ordering
- [ ] Target environment prepared (resource group, networking, identity)
- [ ] Data migration strategy defined (online sync, offline copy, or hybrid)
- [ ] Cutover runbook created with step-by-step instructions
- [ ] Rollback procedure documented with trigger criteria
- [ ] Rollback procedure tested before cutover
- [ ] Downtime window agreed and communicated
- [ ] DNS changes prepared in `infra/dns/subdomain-registry.yaml`
- [ ] Secrets migrated to Azure Key Vault
- [ ] Cutover executed per runbook
- [ ] Functional validation completed (application responds correctly)
- [ ] Performance baseline compared (response times, throughput)
- [ ] Data integrity validated (row counts, checksums, critical records)
- [ ] Source environment retained for rollback period
- [ ] Source decommissioned after validation period
- [ ] Evidence captured in `docs/evidence/{stamp}/caf/migration/`
